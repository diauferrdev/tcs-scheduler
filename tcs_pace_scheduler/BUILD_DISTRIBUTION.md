# Build Distribution Guide

## ⚠️ Important: DO NOT Commit Build Artifacts

**Build outputs (binaries, packages) should NEVER be committed to git.**

- ✅ **Commit**: Source code, scripts, configurations
- ❌ **Don't commit**: `.exe`, `.apk`, `.AppImage`, `.tar.gz`, etc.

All build artifacts are automatically ignored via `.gitignore`.

---

## 🎯 Primary Distribution Strategy

### 📱 Mobile (Android & iOS)
**PRIMARY: Firebase App Distribution**
- Automatic deployment via build scripts
- Testers receive instant notifications
- No manual upload required
- Beta testing and staged rollouts

**Scripts**:
- Android: `bash scripts/build/android.sh patch`
- iOS: `bash scripts/build/ios.sh patch` (requires macOS)

### 🖥️ Desktop (Windows, Linux, macOS)
**PRIMARY: GitHub Releases (Automated)**
- **AUTOMATIC**: Tag-based releases via GitHub Actions
- Push a version tag to trigger automated builds
- All desktop platforms built in parallel
- Artifacts automatically uploaded to GitHub Releases

**How to trigger automated release**:
```bash
git tag v1.0.0
git push --tags
```

**Manual build scripts** (for local testing):
- Windows: `bash scripts/build/windows.sh`
- Linux: `bash scripts/build/linux.sh`
- macOS: `bash scripts/build/macos.sh`

### 🌐 Web
**Nginx Deployment**
- Build static files for web server
- Deploy to Nginx or similar
- Progressive Web App (PWA) support

**Script**: `bash scripts/build/web.sh`

## 🐧 Linux Distribution

| Format | Executable? | Use Case |
|--------|------------|----------|
| **Binary** | ✅ Yes | Direct execution |
| **tar.gz** | ✅ Yes | Download & extract |
| **AppImage** | ✅ Yes | Universal Linux executable |
| **Flatpak manifest** | ❌ No | Flathub Store submission |
| **Snap config** | ❌ No | Ubuntu Store submission |

**Generated by**: `bash scripts/build/linux.sh`

### Usage:

**Direct Download (tar.gz):**
```bash
tar -xzf tcs-pace-scheduler-v1.0.0-linux-x64.tar.gz
cd bundle
./tcs_pace_scheduler
```

**AppImage (Universal):**
```bash
chmod +x TCS-Pace-Scheduler-v1.0.0-x86_64.AppImage
./TCS-Pace-Scheduler-v1.0.0-x86_64.AppImage

# OR (WSL/no FUSE):
./TCS-Pace-Scheduler-v1.0.0-x86_64.AppImage --appimage-extract-and-run
```

**Build Flatpak for Flathub:**
```bash
flatpak-builder build-dir build/linux/flatpak/com.tcs.pace_scheduler.yml
flatpak-builder --repo=repo --force-clean build-dir [manifest]
flatpak build-bundle repo tcs-pace-scheduler.flatpak com.tcs.pace_scheduler
```

**Build Snap for Ubuntu Store:**
```bash
snapcraft
snapcraft upload --release=stable *.snap
```

## Best Practices

1. **Version Control**
   - Only commit source code and scripts
   - Use `.gitignore` to exclude build outputs
   - Tag releases in git: `git tag v1.0.0`

2. **Distribution**
   - Use GitHub Releases for versioned downloads
   - Provide checksums (SHA256) for verification
   - Include installation instructions

3. **Store Submissions**
   - Test thoroughly before submission
   - Follow platform guidelines
   - Include screenshots and descriptions

4. **Automation**
   - Use CI/CD for automatic builds
   - GitHub Actions can build on tag push
   - Firebase App Distribution for beta testing

## 🤖 Automated GitHub Actions Workflow

### Overview
Desktop releases are fully automated via GitHub Actions. When you push a version tag, the workflow:
1. Builds all desktop platforms in parallel (Windows, Linux, macOS)
2. Creates a GitHub Release with the tag name
3. Uploads all build artifacts to the release
4. Uses centralized release notes from `RELEASE_NOTES.md`

### How to Trigger Automated Release

**Step 1: Push a version tag**
```bash
git tag v1.0.0
git push --tags
```

**Step 2: Monitor the workflow**
- Go to GitHub Actions tab in your repository
- Watch the "Build and Release" workflow run
- All builds run in parallel for speed

**Step 3: Check the release**
- Once complete, go to Releases tab
- Your new release will be published with all artifacts

### What Gets Built Automatically

**Windows Build:**
- `tcs-pace-scheduler-vX.X.X-windows-x64.zip` - Portable application

**Linux Build:**
- `TCS-Pace-Scheduler-vX.X.X-x86_64.AppImage` - Universal executable
- `tcs-pace-scheduler-vX.X.X-linux-x64.tar.gz` - Compressed archive
- `com.tcs.pace_scheduler.yml` - Flatpak manifest
- `snapcraft.yaml` - Snap configuration

**macOS Build (when configured):**
- `tcs-pace-scheduler-vX.X.X-macos.zip` - macOS application bundle

### Workflow File Location
`.github/workflows/release.yml`

### Requirements
- Repository must have GitHub Actions enabled
- Tag must match pattern: `v*.*.*` (e.g., v1.0.0, v1.2.3)
- For macOS builds: macOS runner available (currently commented out)
